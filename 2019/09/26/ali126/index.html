阿里蚂蚁金服面试题

Posted on 2019-09-26

阿里蚂蚁金服面试题汇总。

1、Java事件机制包括哪三个部分？分别介绍。

  • 事件：封装事件源对象。

  • 事件监听器：监听事件源的属性。

  • 事件源：事件发生的地方，需要有盛装监听器的容器。

2、为什么要使用线程池？

  • 提高效率：不用对每个请求都创建一个线程。创建一定数量的线程放在线程之中，需要的时候从池中取出。

  • 方便管理：对线程进行统一管理，多出来的请求可以排队等候，避免无休止的创建线程导致系统崩溃。

3、说说几种常见的线程池及使用场景。

  • newFixedThreadPool（固定大小的线程池）

    特点：核心线程数和最大线程数大小一样。

    使用场景：适用于处理CPU密集型的任务。

  • newSingleThreadExecutor（单线程线程池，使用唯一工作线程执行任务）

    特点：核心线程数和最大线程数大小一样且都是1。

    适用场景：适用于串行执行任务的场景。

  • newCachedThreadPool（可缓存线程的线程池，可灵活回收空闲线程）

    特点：核心线程数为0，且最大线程数为Integer.MAX_VALUE。

    适用场景：用于并发执行大量短期的小任务。

  • newScheduledThreadPool（定长线程池，支持周期性事务执行）

    特点：最大线程数为Integer.MAX_VALUE。
    阻塞队列是DelayedWorkQueue。

    适用场景：需要多个后台线程执行周期任务，同时需要限制线程数量的场景。

4、线程池都有哪几种工作队列？

  • ArrayBlockingQueue

    基于数据结构的有界阻塞队列，按FIFO原则堆元素排序。

  • LinkedBlockingQueue

    基于链表结构的阻塞队列，吞吐率高于ArrayBlockingQueue。

  • SynchronousQueue

    一个不存储元素的阻塞队列，吞吐量高于LinkedBlockingQueue。

  • PriorityBlockingQueue

    一个具有优先级的无限阻塞队列。

5、怎样理解无界队列和有界队列？

  • 有界队列有固定大小的队列，例如：设定了大小的LinkedBlockingQueue，SynchronousQueue。

    初始的poolSize < corePoolSize，提交的runnable任务，会直接做为new一个Thread的参数，立马执行 。

    当提交的任务数超过了corePoolSize，会将当前的runable提交到一个block queue中。

    有界队列满了之后，如果poolSize < maximumPoolsize时，会尝试new 一个Thread的进行救急处理，立马执行对应的runnable任务。

    如果无法处理，就会执行reject操作。

  • 无界队列没有设置固定大小的队列，例如：没有设置大小的LinkedBlockingQueue。

    与有界队列相比，除非系统资源耗尽，否则无界的任务队列不存在任务入队失败的情况。

    当有新的任务到来，系统的线程数小于corePoolSize时，则新建线程执行任务。

    当达到corePoolSize后，就不会继续增加，若后续仍有新的任务加入，而没有空闲的线程资源，则任务直接进入队列等待。

    当线程池的任务缓存队列已满并且线程池中的线程数目达到maximumPoolSize，如果还有任务到来就会采取任务拒绝策略。

6、线程池中集中重要的参数及流程说明。

  • corePoolSize

    核心池的大小，当线程池中的线程数目达到corePoolSize后，就会把到达的任务放到缓存队列中。

  • maximumPoolSize

    线程池最大线程数，表示在线程池中最多能创建多少个线程。

  • keepAliveTime

    表示线程没有任务执行时最多保持多长时间就会停止。

  • unit

    参数keepAliveTime的时间单位。

  • workQueue

    一个阻塞队列用来存储等待执行的任务，阻塞队列选择见问题4。

  • ThreadFactory

    用于设置创建线程的工厂

  • handler

    设置拒绝处理任务时的策略。

7、什么是反射机制？

  • Java 反射机制是在运行状态中，对于任意一个类，都能够获得这个类的所有属性和方法，对于任意一个对象都能够调用它的任意一个属性和方法。
